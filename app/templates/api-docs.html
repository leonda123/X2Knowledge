API文档

本工具提供了REST API接口，可以通过程序调用实现文档转换功能。

文档转文本API

POST /api/convert

内容类型: multipart/form-data

请求参数

参数名 类型 必填 描述
file 文件 是 要转换的文档文件

支持的文件类型

  • Word文档 (.doc, .docx)
  • Excel表格 (.xls, .xlsx)
  • PowerPoint演示文稿 (.ppt, .pptx)
  • PDF文档 (.pdf)
  • 文本文件 (.txt)
  • Markdown文件 (.md)

响应格式

成功响应 (HTTP 200):

{
  "text": "转换后的文本内容",
  "filename": "原始文件名",
  "file_size": "文件大小(字节)",
  "processing_time": "处理时间(秒)"
}

错误响应 (HTTP 4xx/5xx):

{
  "error": "错误信息",
  "details": "详细错误信息(如有)"
}

文档转Markdown API

POST /api/convert-to-md

内容类型: multipart/form-data

请求参数

参数名 类型 必填 描述
file 文件 是 要转换的文档文件

支持的文件类型

  • Word文档 (.doc, .docx)
  • Excel表格 (.xls, .xlsx)
  • PowerPoint演示文稿 (.ppt, .pptx)
  • PDF文档 (.pdf)
  • 文本文件 (.txt)
  • Markdown文件 (.md)
  • 音频文件 (.mp3, .wav) - 新增

响应格式

成功响应 (HTTP 200):

{
  "text": "转换后的Markdown内容",
  "filename": "原始文件名",
  "file_size": "文件大小(字节)",
  "processing_time": "处理时间(秒)"
}

错误响应 (HTTP 4xx/5xx):

{
  "error": "错误信息",
  "details": "详细错误信息(如有)"
}

特点

与普通文本转换不同，Markdown转换将保留更多的原始文档结构：

  • 保留标题层级结构
  • 保留列表格式
  • 保留表格结构
  • 保留超链接
  • 对音频文件提取元数据，包括时长、声道数和采样率

Docling文档转Markdown API

POST /api/convert-to-md-docling

内容类型: multipart/form-data

请求参数

参数名 类型 必填 描述
file 文件 是 要转换的文档文件

支持的文件类型

  • Word文档 (.doc, .docx)
  • Excel表格 (.xls, .xlsx)
  • PowerPoint演示文稿 (.ppt, .pptx)
  • PDF文档 (.pdf) - 对PDF有优化的表格识别和VLM支持
  • 文本文件 (.txt)
  • Markdown文件 (.md)
  • CSV文件 (.csv)
  • HTML/XHTML文件 (.html, .xhtml)
  • 图像文件 (.png, .jpg, .jpeg, .tiff, .bmp)
  • 音频文件 (.mp3, .wav)

响应格式

成功响应 (HTTP 200):

{
  "text": "转换后的Markdown内容",
  "filename": "原始文件名",
  "file_size": "文件大小(字节)",
  "processing_time": "处理时间(秒)",
  "converter": "docling"
}

错误响应 (HTTP 4xx/5xx):

{
  "error": "错误信息",
  "details": "详细错误信息(如有)"
}

Docling特点

Docling是一个强大的文档转换工具，针对PDF文档有特殊优化：

  • 使用表格识别技术，更准确地提取PDF中的表格结构
  • 利用视觉语言模型(VLM)处理复杂的PDF布局
  • 支持从图片中提取结构化内容
  • 保留文档的标题层级、列表和表格结构
  • 支持更广泛的输入文件格式

注意: Docling转换器性能最佳需要CUDA环境，当前使用CPU加速模式。它对PDF文件格式有特殊优化，在处理复杂PDF时效果比MarkItDown更好。

批量文件夹转换API

注意: 此API用于批量转换文件夹中的所有支持格式文件，会生成日志文件记录转换过程。

批量转为文本API

POST /api/convert-folder

内容类型: application/json

请求参数

参数名 类型 必填 描述
source_folder String 是 要转换的源文件夹路径(绝对路径)
output_folder String 否 转换结果的输出文件夹路径，如不提供将在源文件夹创建output_text子文件夹

请求示例

{
  "source_folder": "/path/to/documents",
  "output_folder": "/path/to/output"
}

响应格式

成功响应 (HTTP 200):

{
  "source_folder": "/path/to/documents",
  "output_folder": "/path/to/output",
  "log_file": "/path/to/output/conversion_log.txt",
  "total_time": 45.23,
  "results": {
    "total_files": 10,
    "converted_files": 8,
    "skipped_files": 1,
    "failed_files": 1,
    "conversion_details": [
      {
        "file": "/path/to/documents/example.docx",
        "status": "success",
        "output": "/path/to/output/example.txt",
        "processing_time": 5.32
      },
      {
        "file": "/path/to/documents/image.jpg",
        "status": "skipped",
        "reason": "不支持的文件格式: .jpg"
      },
      {
        "file": "/path/to/documents/corrupted.pdf",
        "status": "failed",
        "error": "无法读取文件"
      }
    ]
  }
}

批量转为Markdown API (Docling)

POST /api/convert-to-md-docling-folder

内容类型: application/json

请求参数

参数名 类型 必填 描述
source_folder String 是 要转换的源文件夹路径(绝对路径)
output_folder String 否 转换结果的输出文件夹路径，如不提供将在源文件夹创建output_docling子文件夹

批量转换特点

批量转换API提供以下功能：

  • 递归处理指定文件夹及其子文件夹中的所有文件
  • 在输出文件夹中保持原始文件夹结构
  • 自动跳过不支持的文件格式
  • 生成详细的转换日志记录
  • 提供转换统计和详细信息

Python示例 - 批量转换文件夹

import requests
import json

url = "http://127.0.0.1:5000/api/convert-folder"  # 或者 "/api/convert-to-md-folder" 或 "/api/convert-to-md-docling-folder"
data = {
    "source_folder": "D:/documents",
    "output_folder": "D:/output"
}

headers = {
    'Content-Type': 'application/json'
}

response = requests.post(url, data=json.dumps(data), headers=headers)
result = response.json()

print(f"转换完成，总文件数: {result['results']['total_files']}")
print(f"成功转换: {result['results']['converted_files']} 文件")
print(f"日志文件: {result['log_file']}")

示例代码

Python示例 - 转为文本

import requests

url = "http://127.0.0.1:5000/api/convert"
file_path = "example.docx"  # 替换为您的文件路径

with open(file_path, "rb") as f:
    files = {"file": (file_path, f)}
    response = requests.post(url, files=files)

if response.status_code == 200:
    result = response.json()
    print("转换成功!")
    print(f"文本内容: {result['text'][:100]}...")  # 打印前100个字符
    print(f"文件名: {result['filename']}")
    print(f"文件大小: {result['file_size']} 字节")
    print(f"处理时间: {result['processing_time']} 秒")
else:
    print(f"转换失败: {response.json()['error']}")
